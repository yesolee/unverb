# 06. 로드맵

**형식**: Now / Next / Later
**기준**: 1인 개발

## Now: MVP 벌리드 (~12주)

### Phase 0: 콘텐츠 DB 구축 (2주)

| 항목 | 완료 조건 |
|------|----------|
| 논문 수집 (Semantic Scholar/OpenAlex) | 50편 확보, 카테고리별 분류 |
| 관찰 미션 15개 | 각 미션에 DOI 연결 |
| 탐색 미션 15개 | 돈 안 들고 30분 이내 조건 충족 |
| 성찰 질문 30개 | 탭 선택지 2~3개씩 포함 |
| 안전성 검증 | 금지 표현 0건 |

의존성: 없음 (가장 먼저 시작 가능)

### Phase 1: 핵심 기능 (4주)

| 항목 | 완료 조건 |
|------|----------|
| 미션 화면 | 관찰/탐색 구분 표시 |
| 그림일기 기록 | 사진 + 텍스트 저장/로드 |
| AI 피드백 | Claude API 연동, 담임선생님 톤 |
| 푸시 알림 | FCM 연동, 3개 알림 정시 도착 |
| 기본 인증 | 회원가입 + 로그인 |

의존성: Phase 0 완료, Claude API 키

### Phase 2: 주간 리뷰 + 안전 (3주)

| 항목 | 완료 조건 |
|------|----------|
| 한 주 정리 | Part A+B-1+B-2+Closing, 2분 이내 |
| 동사 발견 로직 | 최소 1개 동사 후보 제시 |
| 안전 시스템 | Level 3 트리거 시 100% 위기상담 연결 |
| 출처 시스템 | 모든 미션에 출처 연결 |
| AI 응답 금지 | 금지 목록 표현 0건 통과 |

의존성: Phase 1 완료 (기록 데이터 필요)

### Phase 3: QA + 베타 (2주)

- 내부 QA: 전체 플로우 + 엣지 케이스 + 안전 시스템 테스트
- 베타 테스트: 지인 5~10명에게 1주일 사용 후 피드백
- 버그 수정 + 성능 최적화

### Phase 4: 출시 (1주)

- 앱스토어 제출 (스크린샷, 설명, 개인정보처리방침)
- 랜딩 페이지
- 출시

## Next: MVP 후 개선 (1~3개월)

출시 후 데이터를 보고 판단. PMF 확인이 최우선.

| 항목 | 조건 |
|------|------|
| 대시보드 | 사용자가 동사 발견 후 "모아보고 싶다"는 요구 |
| 성찰 질문 확장 | 미션 완료율 60%+ 이고 "더 깊이" 원할 때 |
| 온보딩 플로우 | Day 1 이탈률이 높을 때 |
| 콘텐츠 추가 | 미션 30개 소진 시점 전에 |
| 오후 2시 체크인 | 미션 완료율 40% 미만일 때 |

## Later: 스케일업 (3~6개월+)

PMF가 확인된 후에만 고려.

| 항목 | 조건 |
|------|------|
| RAG 기반 개인화 | 30일 유지율 20%+ 확인 후 |
| 커뮤니티 | 사용자가 "다른 사람의 동사도 보고 싶다" 요구 시 |
| 수익 모델 (프리미엄) | MAU 1만+ 도달 후 |
| 전문가 자문단 | MAU 10만+ 도달 후 |
| 다국어 지원 | 한국 시장 PMF 확인 후 |

## 의존성 맵

```
Phase 0: 콘텐츠 DB
    │
    ▼
Phase 1: 핵심 기능 ← Claude API 키 필요
    │
    ▼
Phase 2: 주간 리뷰 + 안전
    │
    ▼
Phase 3: QA + 베타
    │
    ▼
Phase 4: 출시 ← Apple Dev Account 필요
    │
    ▼
[MVP 출시] → Next → Later
```

## 출시 후 판단 기준

| 신호 | 의미 | 액션 |
|------|------|------|
| 7일 유지율 < 20% | 컨셉 자체에 문제 | 사용자 인터뷰 → 컨셉 재검토 |
| 유지율 OK + 미션 완료율 < 40% | 미션이 어렵거나 동기 부족 | 미션 난이도 조정 + 의미 텍스트 개선 |
| 유지율 OK + 동사 발견율 < 50% | 주간 리뷰 플로우 문제 | AI 동사 추출 로직 개선 |
| 모든 지표 OK | PMF 신호 | Next 단계로 전환 |
